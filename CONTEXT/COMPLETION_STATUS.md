## 💡 Ключевые решения архитектуры

### Почему 3-слойная архитектура?

**Domain слой** 
- Полностью независим
- Легко юнит-тестировать
- Переиспользуемый в других UI

**Data слой**
- Абстрактный формат (JSON)
- Легко менять на БД позже
- DTO паттерн для безопасности

**Presentation слой**
- Легко менять графику (были Lanterna → теперь LibGDX)
- Не содержит бизнес-логику
- Ответственен только за отображение

### Почему пошаговая система?

- **Проще тестировать** - детерминированная
- **Ясная логика** - ход за ходом
- **Сохранение просто** - состояние между ходами
- **ИИ легко** - враги ходят по очереди
- **Баланс легче** - контролируемые скорости

### Почему JSON сохранения?

- **Читаемо людям** - можно отладить вручную
- **Версионируемо** - git видит изменения
- **Портативно** - работает везде
- **Jackson** - готовая библиотека
- **DTO** - типо-безопасно
